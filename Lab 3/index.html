¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
  • GET: Solicita una representación, en forma de datos, del recurso especificado.
  • HEAD: Solicita una respuesta identica a la solicitud GET pero sin el BODY de la respuesta.
  • POST: Envía una entidad al recurso especificado, comúnmente causando un cambio en el estado del objeto.
  • PUT: Reemplaza todas las representaciones actuales del objeto especificado con los argumentos de la solicitud.
  • PATCH: Aplica modificaciones parciales al objeto.
  • DELETE: Elimina el objeto especificado.
¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué? Para asegurar la seguridad del usuario, se debe utilizar el metodo de POST, porque después de que se haya enviado el forms, los valores introducidos no serán visibles en la barra de dirección de la nueva ventana del buscador.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error? Cuando todo está funcionando como esperado y no ocurrió ningún error
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué? Un error 404 se presenta cuando el usuario trata de ingresar a una URL que no es válida, por lo tanto, el desarrollador tiene la responsalidad de averiguar si el usuario trata de acceder a una página válida con una URL distinta o no
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué? El desarrollador tiene toda la responsabilidad de solucionar el problema porque cuando se presenta un error 500 es cuando se presente una condición desconocida que no permite al programa ejecutar la solicitud que se le pidió
¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados. Un atribtuo depreciado es un elemento que no se recomienda utilizar para los desarrolladores ya que no se encuentra en soporte. Algunos ejemplos son: No se debe usar un tag border en un tag img, no se debe usar un tag language en un tag script, no se debe usar un tag name en un tag a

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS? Mi recomendación es que se use cuando no se está cumpliendo el comportamiento deseado y se tiene la seguridad que el atributo está implementado correctamente, porque lo que hace el comando de important es que le pone más importancia al atributo de CSS que se está marcando como importante. Yo en lo personal usaría el !important como último recurso porque puede sobrescribir algunos de los templates que declaramos anteriormente para la clase o tag en particular.
Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado? Por diferentes motivos:
  1. Si la imagen es de mucha calidad, va a tardar más en cargarse
  2. La imagen puede no ser de la misma resolución del fondo
  3. Si la imagen es muy llamativa, distrae la parte importante de la página, que es el contenido
  4. Los colores de la imagen pueden no ser los mismos que la paleta de colores o del estilo de la página
Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt? Yo daría como recomendación utilizar % y px, ya que los pixeles son las unidades que comúnmente se miden las imágenes, por lo tanto, será más cómodo trabajar con px cuando tenemos una referencia específica a la cual queremos definir nuestra imagen. Por el otro lado, si no tenemos una referencia, podemos usar porcentajes, lo cual también nos ayudará para adecuar la imagen a diferentes resoluciones
¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio? Los archivos CSS que nosotros escribimos tienen mucho espacio vacío, lo cual nos ayuda mucho a nosotros los desarrolladores para entender el código. Sin embargo, si se hace una versión minimizada del CSS, la computadora lo compila y carga más rápido
