¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
  • GET: Solicita una representación, en forma de datos, del recurso especificado.
  • HEAD: Solicita una respuesta identica a la solicitud GET pero sin el BODY de la respuesta.
  • POST: Envía una entidad al recurso especificado, comúnmente causando un cambio en el estado del objeto.
  • PUT: Reemplaza todas las representaciones actuales del objeto especificado con los argumentos de la solicitud.
  • PATCH: Aplica modificaciones parciales al objeto.
  • DELETE: Elimina el objeto especificado.
¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué? ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué? Un error 404 se presenta cuando el usuario trata de ingresar a una URL que no es válida, por lo tanto, el desarrollador tiene la responsalidad de averiguar si el usuario trata de acceder a una página válida con una URL distinta o no
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué? El desarrollador tiene toda la responsabilidad de solucionar el problema porque cuando se presenta un error 500 es cuando se presente una condición desconocida que no permite al programa ejecutar la solicitud que se le pidió
¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados. Un atribtuo depreciado es un elemento que no se recomienda utilizar para los desarrolladores ya que no se encuentra en soporte.
